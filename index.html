我的第一篇博客

在使用 Hexo 搭配 GitHub Pages 搭建并部署个人博客的过程中，会遇到一系列环境配置、连接认证及部署执行相关问题。
###一、本地环境基础配置问题
####Node.js 与 npm 命令识别异常
执行 node -v 或 npm -v 时，若出现
“命令未找到”报错，需优先排查：

  • 环境变量配置：确认 Node.js 安装路径已添加至系统环境变量 Path 。可通过重新安装 Node.js 并选择“自动配置环境变量”简化操作。
  • 终端环境影响：PowerShell 执行策略可能限制脚本运行，切换至 CMD 终端测试命令，避免因终端环境差异导致误判。

###二、GitHub 连接问题
####执行 ssh -T git@github.com 测试时，遇报错 Connection refused ，常见原因及解决

  • 网络端口限制：校园网、企业网可能屏蔽 22 端口，可修改 SSH 配置强制走 443 端口：
  1. 在 .ssh 目录新建/编辑 config 文件，添加：
    Host github.com
    Hostname ssh.github.com
    Port 443
  2. 重新执行 ssh -T git@github.com ，按提示输入 yes 确认连接。
  • 公钥配置不完整：检查 GitHub 后台公钥内容，确保粘贴的是 id_rsa.pub 完整文本，无多余空格或换行。

###三、Hexo 部署关键配置与执行问题
安装 Hexo 时遇到的问题有：

  1. 命令未识别： hexo 、 git 命令因环境变量未配置或配置错误，系统无法找到执行路径，提示不是内部/外部命令 。
  2. 权限不足：执行 hexo init 时，因命令行权限不够，无法在 D 盘创建目录，报错 “EPERM: operation not permitted” 。
    3.部署命令适配：PowerShell 不支持 && 连续执行语法
    解决方法：
  3. 找到 npm 或 Git 安装路径，将其可执行文件所在目录添加到系统环境变量 PATH 中，重启命令行验证。
  4. 以管理员身份运行命令行工具；或检查文件夹权限，赋予当前用户完全控制、写入等权限，再执行相关命令。
  5. 改用 CMD 终端执行hexo clean && hexo generate && hexo deploy 或拆分命令分步执行：hexo clean
    hexo generate
    hexo deploy

###四、总结
从本地环境配置到 GitHub 连接认证，再到 Hexo 部署调试，需严格遵循流程并关注细节。遇到报错时，优先通过终端报错信息定位问题（如网络端口、命令语法、文件路径等）。尽管过程遇到了多环节问题，但成功部署后通过 https://用户名.github.io 访问到个人博客的体验，能有效验证技术实践价值。
